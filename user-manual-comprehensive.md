---## Chapter 1: Introduction

### Overview of AKC Smart Campus

The AKC Smart Campus (also known as CampusWise Smart Hub) is a comprehensive integrated resource management system designed specifically for university campuses. This platform consolidates multiple operational functions into a single, centralized hub, enabling seamless coordination across administrative, academic, and facility management domains.

### System Purpose and capabilities

The system serves as a unified digital backbone for campus operations, providing:

- **Venue and Resource Booking Management**: Centralized scheduling of classrooms, meeting rooms, equipment, and other campus facilities with real-time availability tracking and conflict resolution
- **Energy Management and Monitoring**: Real-time consumption tracking, predictive analytics, anomaly detection, and recommendations for efficiency improvements across all campus buildings and blocks
- **Water Conservation Monitoring**: Comprehensive water usage tracking, location-specific analytics, automated alerts for unusual patterns, and sustainability reporting
- **HR and Workload Management**: Staff roster management, workload balancing, stress and burnout indicators, approval workflows, and comprehensive audit trails
- **Integrated Approvals Workflow**: Centralized approval system for bookings, training, feedback, and other operational requests with status tracking and audit logs
- **Administrative Functions**: System administration, user management, backup and restore capabilities, log monitoring, and security controls

### Target Users

This manual is designed for:
- **Faculty Members**: Booking rooms and resources for academic activities
- **Administrative Staff**: Managing approvals, HR functions, and operational workflows
- **Facilities Managers**: Monitoring energy and water consumption, managing maintenance schedules
- **System Administrators**: Managing users, backups, configuration, and system health

### System Architecture

The AKC Smart Campus operates on a role-based access control (RBAC) architecture with 7 permission levels, from Viewer (read-only) to Super Auditor (full audit visibility). This ensures that each user has appropriate access to perform their duties while maintaining security and audit compliance.

---## Chapter 3: Authentication and Login

### Login Process

The login process is the first interaction with the AKC Smart Campus system. It validates your identity and establishes a secure session.

#### Accessing the Login Page

1. **Open Web Browser**: Use Chrome, Firefox, Safari, or Edge
2. **Navigate to URL**: Type http://10.67.232.58:8080
3. **System Resolves**: Login page loads automatically if not authenticated
4. **Pre-Login Display**: If already logged in, redirects to dashboard

#### Completing Login Form

The login form requests:
- **Email Address**: Your institutional email (e.g., user@apollo-university.edu)
- **Password**: Your institutional password set with IT
- **Remember Me**: Optional checkbox to skip login next time on same device

#### Submitting Login

1. **Enter Email**: Type institutional email address
2. **Enter Password**: Type password (characters display as dots for security)
3. **Click Login**: Submit credentials for authentication
4. **Validation**: System checks credentials against institutional directory
5. **Success**: Redirected to your personalized dashboard
6. **Error**: Error message displays if credentials incorrect

#### Understanding Login Errors

| Error Message | Cause | Solution |
|---|---|---|
| Invalid email or password | Wrong email or password | Verify both and retry |
| Email not found | Not an institutional account | Contact IT to verify account |
| Account locked | 5 failed login attempts | Wait 15 minutes and retry |
| Session already active | Logged in on another device | End other session or use "Use This Device" option |
| Server error | System temporarily unavailable | Wait and retry after 5 minutes |

### Forgot Password

If you forget your password:

1. **Click "Forgot Password"** on login page (below password field)
2. **Enter Email Address**: Your institutional email
3. **Submit**: System sends reset link
4. **Check Email**: Look for email from noreply@campuswise.edu
5. **Click Reset Link**: Opens password reset form
6. **Enter New Password**: Must meet complexity requirements
7. **Confirm Password**: Retype to verify match
8. **Submit**: System updates password
9. **Log In**: Use new password to log in

#### Password Reset Tips

- **Email Delay**: Reset email may take up to 5 minutes to arrive
- **Check Spam**: If email doesn't appear, check spam/junk folder
- **Link Expiry**: Reset link expires after 30 minutes
- **One-Time Use**: Each reset link works only once
- **New Password Required**: Previous password no longer works

### Multi-Device Login

The system restricts one active session per user:

- **First Device**: Successfully logs in
- **Second Device**: When you log in on different device/browser
- **Session Conflict**: "Session conflict detected" message appears
- **Choose Device**: Select which device should remain active
- **Other Devices**: Automatically logged out from other devices

#### Managing Multiple Logins

If you see session conflict:

1. **Review Message**: Shows where other active session is
2. **Choose Action**:
   - "Use This Device": Stay logged in here, log out other location
   - "Use Other Device": Go back to that device
3. **Confirmation**: Selected device remains logged in
4. **Other Sessions**: Terminated automatically

**Note**: This security feature prevents unauthorized session hijacking.

### Session Management

Once logged in, your session:
- **Expires After**: 8 hours of no user activity (clicks, typing, scrolling)
- **Warning**: 15-minute warning before expiration
- **Extend Session**: Click "Continue Session" in warning dialog
- **Automatic Refresh**: Session extends automatically while actively using system
- **Maximum Duration**: Absolute 24-hour limit regardless of activity

#### Session Expiration Warning

When approaching expiration:

1. **Warning Dialog**: Modal appears on screen
2. **Time Remaining**: Shows "Session expires in 15 minutes"
3. **Action Required**: Either continue session or log out
4. **Continue Session**:
   - Click "Continue Session" button
   - Session extends for another 8 hours
   - No need to re-enter password
5. **Let Expire**:
   - Click "Log Out" button
   - Or simply close dialog
   - Session ends, must log in again

#### After Session Expires

If session expires without action:
- **Red Warning**: "Your session has expired" message displays
- **Automatic Redirect**: Redirected to login page
- **Any Unsaved Work**: Lost when session expires
- **Best Practice**: Save work frequently to avoid data loss

### Session Security

Security features protecting your session:

- **Encrypted Connection**: All communication over HTTPS (padlock icon in browser address bar)
- **Token Validation**: Server validates session token on every request
- **IP Monitoring**: Suspicious IP changes monitored
- **Device Binding**: Session bound to device/browser
- **No Sharing**: Session cannot be shared or transferred

### Login Troubleshooting

#### Most Common Login Issues

**Problem: Forgot Email Address or Username**
- Solution: Check welcome email from IT or institutional directory
- Alternative: Use office phone number or ask colleague
- Prevention: Write down institutional email in secure location

**Problem: Reset Password Link Expired**
- Solution: Go back to login and select "Forgot Password" again to get new link
- Note: Links valid for 30 minutes only
- Action: Don't wait too long between clicking link and using it

**Problem: Computer Remembers Password Incorrectly**
- Solution: Clear saved password in browser settings
- For Chrome: Settings > Passwords > Remove saved password
- For Firefox: Tools > Options > Privacy > Saved Logins > Remove

---## Chapter 4: Understanding Your Role

### Role-Based Access Control (RBAC)

The AKC Smart Campus uses a 7-tier permission system to ensure users have appropriate access for their responsibilities while maintaining security. Your role determines:
- Which modules you can access
- Which records you can view or edit
- What actions you can perform
- What reports you can generate
- Which administrative functions are available

### The 7 Permission Levels

#### Level 7: Super Auditor

**Purpose**: Full compliance and audit oversight

**Permissions**:
- View all system data (read-only)
- Access complete audit trails
- View user activity logs
- Cannot modify any data

**Who**: Compliance officers, auditors, security team

**Typical Functions**:
- Compliance reporting and verification
- Security event investigation
- Audit trail review for regulatory requirements
- User activity monitoring

---

#### Level 6: Ops Manager

**Purpose**: Cross-module operational management

**Permissions**:
- Access and report on all operational data
- View real-time dashboards
- Generate comprehensive reports
- Oversee approvals workflows
- Limited administrative functions

**Who**: Operational managers, operations coordinators

**Typical Functions**:
- Monitor system performance
- Generate operational reports
- Escalate approval items
- Coordinate between modules

---

#### Level 5: Super Admin

**Purpose**: System-wide configuration and governance

**Permissions**:
- Configure system settings
- Manage all users and roles
- Create and modify templates
- Access all configuration modules
- View complete data dictionary

**Who**: System administrators, IT managers

**Typical Functions**:
- User provisioning and deprovisioning
- Role and permission management
- System configuration
- Template management
- Global settings

---

#### Level 4: Admin

**Purpose**: Module-specific administration

**Permissions**:
- Administer specific modules (Bookings, Energy, Resources, etc.)
- Manage module-level settings
- Make approval decisions
- Generate module reports
- Manage module-specific users and data

**Who**: Department administrators, module managers

**Typical Functions**:
- Approving requests in assigned module
- Configuring module settings
- User management within module
- Data governance for module

---

#### Level 3: Manager

**Purpose**: Department-level resource management

**Permissions**:
- Manage department/team data
- Make approvals for team members
- View team workload and assignments
- Generate department reports
- Limited user management (own team)

**Who**: Department managers, team leads

**Typical Functions**:
- Team workload management
- Approving team requests
- Department reporting
- Staff oversight

---

#### Level 2: Faculty/Worker

**Purpose**: Operational data entry and request submission

**Permissions**:
- Create bookings and requests
- Edit own records
- Submit approval requests
- View personal data and activity
- Limited reporting (own data)

**Who**: Faculty, staff, regular employees

**Typical Functions**:
- Creating venue bookings
- Submitting requests
- Time and activity entry
- Personal profile management

---

#### Level 1: Viewer

**Purpose**: Information access without modification

**Permissions**:
- View assigned data (read-only)
- Access assigned dashboards
- View reports
- No creation or modification rights

**Who**: Consultants, temporary staff, observers

**Typical Functions**:
- Dashboard viewing
- Report viewing
- Data observation

### Finding Your Role

To see your current role:

1. **Click Profile Icon**: Top-right corner near your name
2. **Select Settings**: Dropdown menu option
3. **View Profile**: My Profile section shows your role
4. **Verify Access**: Check if expected modules are visible in sidebar

### Requesting Role Changes

If your role needs to change:

1. **Contact Your Manager**: Discuss role requirements
2. **Manager Submits Request**: To system administrator
3. **Admin Reviews**: Verifies role is appropriate
4. **Role Updated**: Changes typically apply within 1 hour
5. **Confirm Access**: Check that new permissions visible

### Note About Your Permissions

### Security Overview

The AKC Smart Campus implements comprehensive security measures to protect institutional data, ensure system integrity, and maintain compliance with educational and financial regulations. Security is built into every layer of the system from user authentication to data transmission.

### Authentication and Authorization

#### User Authentication
The system uses multiple levels of authentication:

1. **Primary Authentication**: Institutional email and password
   - Credentials validated against institutional directory
   - Passwords must meet complexity requirements
   - Failed login attempts are monitored and restricted after 5 attempts

2. **JWT (JSON Web Tokens)**: Session Management
   - Tokens expire after 8 hours of inactivity
   - Tokens are refreshed automatically when active
   - Only one active session per user (prevents simultaneous logins)
   - Token includes role and permission information

3. **Enhanced Security** (Optional for sensitive users):
   - Two-factor authentication (2FA) via email or authenticator app
   - IP restriction for specific user types
   - Session binding to device/browser

#### Authorization and Role-Based Access Control

The system implements strict RBAC:

| Role Level | Role | Scope | Typical Users |
|---|---|---|---|
| 7 | Super Auditor | Full visibility | Compliance officers, auditors |
| 6 | Ops Manager | Cross-module operations | Operational managers |
| 5 | Super Admin | Global configuration | System administrators |
| 4 | Admin | Module administration | Department admins |
| 3 | Manager | Department level | Department managers |
| 2 | Faculty/Worker | Operational access | Faculty and staff |
| 1 | Viewer | Read-only | Limited access users |

**How Authorization Works**:
- Each feature, page, and data record has defined access requirements
- System checks user's role before allowing access
- Users see only modules and data they're authorized for
- Attempts to access unauthorized resources are logged as security events

### Common Issues and Solutions

#### Login and Authentication Issues

**Issue: Cannot Log In - Credentials Rejected**
- **Error Message**: "Invalid email or password"
- **Possible Causes**: Incorrect password, email not in system, account locked
- **Solutions**:
  1. Verify email address is spelled correctly
  2. Check if CAPS LOCK is on
  3. Use "Forgot Password" to reset password
  4. Wait 15 minutes if account is locked (after 5 failed attempts)
  5. Contact IT if problem persists
- **Prevention**: Write down your institutional email in secure location

**Issue: Session Conflict Warning**
- **Error Message**: "Your session is active on another device"
- **Possible Causes**: Logged in on multiple devices simultaneously
- **Solutions**:
  1. Click "Use This Device" to end other session
  2. If you don't recognize the other device, change password
  3. Check "Active Sessions" in Security Settings
  4. Contact IT if suspicious access suspected
- **Prevention**: Always log out before leaving computers

**Issue: Session Expired Without Warning**
- **Error Message**: "Your session has expired, please log in again"
- **Possible Causes**: Idle longer than 8 hours, browser closed unexpectedly
- **Solutions**:
  1. Log in again with your credentials
  2. If frequent, may indicate network issues
  3. Contact IT if repeatedly happens during active use
- **Prevention**: Return to dashboard before 15-minute expiry warning

**Issue: Forgot Password - Email Not Received**
- **Error Message**: "We've sent a reset link to your email" but no email arrives
- **Possible Causes**: Email in spam folder, incorrect email configured, IT issue
- **Solutions**:
  1. Check spam/junk folder for reset email
  2. Whitelist noreply@campuswise.edu in email filters
  3. Wait 10 minutes (sometimes delayed)
  4. Verify email address in system is correct
  5. Contact IT to manually reset password
- **Timeline**: Email should arrive within 5 minutes

#### Application Performance Issues

**Issue: Page Loads Very Slowly**
- **Error Message**: Page keeps loading (spinning wheel) for extended time
- **Possible Causes**: Network issues, server overload, browser cache, too many tabs
- **Solutions**:
  1. Check internet connection speed
  2. Close unnecessary browser tabs
  3. Clear browser cache and cookies
  4. Try different browser (Chrome, Firefox, Safari)
  5. Try on different computer if available
  6. Check if IT has known outages posted
- **Timeline**: Most pages should load within 5 seconds

**Issue: Features Not Loading - Blank Page Sections**
- **Error Message**: Page loads but certain areas show blank or incomplete
- **Possible Causes**: JavaScript disabled, outdated browser, cache issue
- **Solutions**:
  1. Verify JavaScript is enabled in browser
  2. Update browser to latest version
  3. Clear browser cache: Ctrl+Shift+Delete
  4. Close and reopen browser completely
  5. Try in incognito/private mode
  6. Try different browser
- **Browser Requirements**: Chrome/Firefox/Safari/Edge 2020 or newer

**Issue: Timeouts When Processing Large Requests**

  1. Reduce date range (try smaller time period)

  3. Try generating during off-peak hours

#### Energy and Water Module Issues

**Issue: No Data Showing in Energy/Water Dashboard**
- **Error Message**: Charts are blank, metrics show no values
- **Possible Causes**: No meters configured, data not yet available, permission issue
- **Solutions**:
  1. Verify meters are installed and transmitting
  2. Contact Facilities/Energy team to check
  3. Check if you have permission to view data
  4. Try specific meter vs. building dashboard
  5. Contact IT with building/meter IDs
- **Data Lag**: Real-time data updates every 30 seconds, historical data may have 1-hour lag

**Issue: Readings Seem Incorrect or Unrealistic**
- **Error Message**: Energy spike, unusual pattern, obviously wrong number
- **Possible Causes**: Sensor error, data transmission issue, system misconfiguration
- **Action Steps**:
  1. Take screenshot showing the anomaly
  2. Note the date, time, and specific metric
  3. Check if issue appears in reports too
  4. Report to Energy Manager and IT
  5. Include meter IDs and building location
- **Investigation**: IT will validate sensor data and system calculations

#### HR and Workload Issues

**Issue: Workload Calculation Seems Wrong**
- **Error Message**: Staff member shows high stress but appears fine
- **Possible Causes**: Workload model configuration, data entry issue, system logic
- **Solutions**:
  1. Review detailed workload metrics
  2. Verify all bookings/assignments are logged
  3. Check if person took leave that's not recorded
  4. Compare with your direct observation
  5. Contact HR if concern about person's status
- **Note**: System recommends follow-up, not diagnostic tool

**Issue: Staff Member Cannot See Their Profile**
- **Error Message**: "Access denied" viewing own HR profile
- **Possible Causes**: Permission not granted, account issue
- **Solutions**:
  1. User should retry from home page
  2. Contact manager to grant profile access
  3. HR can verify permissions are set correctly
  4. May need to wait 1 hour for permissions to apply
